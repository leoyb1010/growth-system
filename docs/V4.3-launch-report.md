## 二、P0 修复清单

### P0-1: Dashboard SQLite 下 Project.year 报错
- **问题**：`dashboardController` 查 `Project.count({ where: { year: currentYear } })`，但 Project 表没有 `year` 字段
- **修复**：
  - `models/index.js`：Project 新增 `year` 字段（INTEGER, NOT NULL, DEFAULT 2026）
  - SQLite：`ALTER TABLE projects ADD COLUMN year INTEGER NOT NULL DEFAULT 2026`
  - `projectController.js`：create/update 白名单加 `year`
  - `init.sql`：INSERT 语句加 `year` 列
  - 存量数据：`UPDATE projects SET year = 2026`
- **验证**：`GET /api/dashboard` 返回 code:0, effective_quarter: Q2 ✅

### P0-2: Search iLike SQLite 兼容问题 + 全仓库排查
- **问题**：SQLite 不支持 `ILIKE`，3 处 `Op.iLike` 会导致查询报错
- **修复**：
  - `searchController.js`：`Op.iLike` → `Op.like`
  - `projectController.js`：`Op.iLike` → `Op.like`
  - `achievementController.js`：`Op.iLike` → `Op.like`
  - 全仓库搜索确认：0 处 iLike 残留
  - JSONB/ENUM 排查确认：Sequelize 自动映射，无兼容问题
- **验证**：`GET /api/search?q=test` 返回 code:0 ✅

### P0-3: dataScope 未全量落地问题
- **问题**：controller 只用 `req.deptFilter`（部门级），`self` 范围下的 `owner_user_id/creator_id` 过滤未生效
- **修复**：
  - `monthlyTaskController.js`：新增 self 范围过滤（`Op.or: [{creator_id}, {owner_name}]`）
  - `achievementController.js`：同上
  - `exportController.js`：新增 `deptFilter` 过滤，防止越权导出
  - 6 个 controller create/update 全部加字段白名单：
    - monthlyTask（create+update）
    - achievement（create+update）
    - performance（create+update）
    - kpi（update）
- **验证**：dept 账号 API 全部返回 code:0 ✅
